## Remaining (intentionally out of scope)
- `bot/src/lib/ai.ts` still contains many `any` typings (`ctx: any`, `messages: any[]`, etc.). Tightening these requires a non-trivial refactor of the streaming/tool-call control flow and risks runtime regressions; the bot workspace has no `lint` script, so these do not produce CI warnings. Leaving for a dedicated follow-up.
- `Tool.function: (args: any)` retains a narrowly-scoped `eslint-disable` directive because TS function-parameter contravariance prevents typed tool implementations (e.g. `({ url, method }) => ...`) from assigning to a `Record<string, unknown>` parameter slot.
- Three `state_referenced_locally` warnings in `webapp/src/routes/+page.svelte` (lines 22, 23, 25) predate this work and concern `$state` initialization patterns, not type safety.
